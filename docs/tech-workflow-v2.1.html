01 整体架构

Thailand10 是一个零服务器、零运营成本的自动化新闻简报系统。除 AI API 调用费用外，所有基础设施完全免费。

核心设计原则（v2.1）
层级技术成本
调度触发OpenClaw Cron（Mac Mini 后台）× 3 个 job免费
Ingest 主控minimax sub-agent（Orchestrator）~$0.0004/次
过滤 + 去重scanner sub-agent（Gemini Flash）~$0.005/次
翻译 + 标注Python → OpenRouter API（scanner JSON mode）~$0.010/次
数据入库Python pool_merge.py免费
发布主控minimax sub-agent（Publish Prompt）~$0.001/次
页面渲染Python build_html.py（无第三方依赖）免费
网站托管GitHub Pages（全球 CDN）免费

📊 每日 Ingest 合计 ~$0.015，每月 ~$0.45（vs v1 每月 $9–12）

💡 架构思路
AI 只做一件事：编辑判断。其余所有工作（格式化、存储、发布）都是确定性的管道，不依赖 AI，可靠、可复现、成本可控。
02 触发层：OpenClaw Cron

OpenClaw Gateway 是运行在 Mac Mini 上的后台守护进程，维护3个 Cron 任务：

任务名时间Prompt 文件模型
每日 Ingest每天 08:30 BKKexperiment/prompts/orchestrator.mdminimax
Thailand10 周四每周四 09:30 BKKexperiment/prompts/publish.mdminimax
Thailand10 周一每周一 09:30 BKKexperiment/prompts/publish.mdminimax

Ingest 每天跑，补充素材进 data/news_pool.json；Publish 每周一/四从 pool 选编并发布。两个流程完全解耦。

03 两阶段流水线：Ingest + Publish

🔽 Ingest Pipeline（每天 08:30）

负责将原始新闻素材处理入库，由 minimax Orchestrator 主控，分9步完成：

1
确定日期 + 上次入库时间
读取 data/last_ingest.txt，确定抓取时间窗口
2
双轨抓取原始素材
fetch_rss.py（6个英文 RSS 源）+ fetch_brave.py（11组关键词）→ TODAY-raw.json
3
展平 + Pool 摘录
Python 内联：dict → 统一数组 TODAY-flat.json；取最近10天 pool 前100条 → TODAY-pool-excerpt.json
4
Filter + Dedup（scanner sub-agent）
Layer 1：过滤非泰国内容 → TODAY-filtered.json
Layer 2：与近期 pool 去重（10天窗口）→ TODAY-deduped.json
5
Translation（Python → OpenRouter API）
scripts/translate.py：分批调用 scanner JSON mode，生成 title_cn / summary_cn / importance / section_hint 等字段 → TODAY-translated.json
6
备份 + 入库
cp news_pool.json news_pool.bak.json，然后 pool_merge.py 合并写入 data/news_pool.json
7
Telegram 通知
发送入库统计：条数 / P1-P3 分布 / 各板块数量

📰 Publish Pipeline（周一/四 09:30）

从 pool 中选题编写，由 minimax Publish Agent 执行：

1
从 news_pool.json 选题
按 importance + section_hint 选 15-25 条，参考 editorial_feedback.md 编辑规则
2
撰写正文 + 生成期数 JSON
AI 编辑：扩写 summary_cn，加编辑视角，输出 thailand10/YYYY-MM-DD-NNN.json
3
渲染 HTML + 推送发布
build_html.py → git push → GitHub Pages 上线，Telegram 通知
04 数据层：JSON 文件

所有持久化数据存在 data/ 目录下，用 JSON 文件模拟轻量级数据库。

data/news_pool.json — 核心素材库

Ingest 每天新增的翻译后条目，是 Publish 的唯一原料来源。每条包含完整字段：id、title_cn、summary_cn、importance（P1/P2/P3）、section_hint、tags、expires_date 等。备份文件：data/news_pool.bak.json（每次 ingest 前自动覆盖）。

data/buffer.json — 补充储备库

Publish 选题时如 pool 内容不足，从此处补充。按时效性分类：时效性内容（14天）/ 时效中性内容（90天）。

data/editorial_feedback.md — 编辑规则

人工维护的编辑偏好文件，Publish Agent 选题时参考。记录哪类内容优先、哪类降权、特殊说明等。

data/issues/ — 每日中间产物

文件内容生命周期
TODAY-raw.jsonRSS + Brave 原始抓取永久保留（近期），旧文件归档
TODAY-flat.json展平后统一数组下次 ingest 后可删
TODAY-filtered.jsonLayer 1 过滤结果同上
TODAY-deduped.jsonLayer 2 去重结果同上
TODAY-translated.jsonLayer 3 翻译结果同上（已入 pool）
05 抓取层：fetch_rss.py

纯 Python 标准库实现，无第三方依赖。抓取 6 个英文媒体 RSS 源，每个来源有权重评分。

媒体来源 ID权重特点
Bangkok Post（头条）bangkokpost_top5最权威英文媒体
Bangkok Post（房产）bangkokpost_property5房产专版，精准
The Thaigerthaiger4外籍读者视角，覆盖广
Khaosod Englishkhaosod4泰文主流媒体英文版
The Nationnation4英文严肃媒体
Pattaya Mailpattaya_mail4芭提雅专属

脚本执行流程：

  1. HTTP GET 每个 RSS XML，User-Agent 标注为 Bangkok-News-Bot/1.0
  2. 解析 <item> 条目，过滤 N 天内的内容（周四刊4天，周一刊含周末4天）
  3. 用 md5(title + url) 生成 12 位哈希作为唯一 ID
  4. 剥离 HTML 标签，提取纯文字摘要（最多600字）
  5. 跨源去重（同一条新闻被多个源收录时只保留一条）
  6. 按发布日期倒序排列，输出标准化 JSON
⚠️ RSS 只有摘要
RSS 条目通常只包含标题和前几句描述，不含全文。Sub-agent 如需撰写详细摘要，还会调用 web_fetch 工具读取原文页面。来源未经人工核实的内容一律以原文为准，不得依赖摘要推断。

Brave Search 补充搜索（11 组关键词）覆盖 RSS 未收录的内容，包括泰文搜索（เศรษฐกิจไทย ล่าสุด等）和特定地产关键词。中泰板块为触发式，只在有重磅新闻时才搜索。

06 渲染层：build_html.py

纯 Python 标准库，无第三方依赖。输入一个期数 JSON，输出一个完整 HTML 页面。

# 用法
python3 scripts/build_html.py data/issues/2026-02-26.json

# 输出
# → thailand10/2026-02-26.html   （期刊正文页）
# → thailand10/index.html         （归档列表，自动插入新条目）

五板块配置

板块定义在脚本顶部的 SECTIONS 列表里，修改这里即可调整所有期刊的板块显示：

板块 ID图标显示名配额
thailand📡政经动态无上限，兜底板块
property🏠房地产动态，可为 0
bangkok🛺曼谷动态，严格本地新闻
pattaya🌅芭提雅动态，可为 0
cn_thai🚅中泰触发式，可为 0

空板块自动跳过——section_html() 函数检测到空数组直接返回空字符串，不渲染板块标题，保持页面整洁。

样式分离

所有样式存在独立的 assets/style-thailand10.css，HTML 文件只引用外部 CSS。修改样式不需要重新生成所有历史页面——浏览器每次加载时都拉取最新 CSS。

07 👍🏻👎🏻 反馈机制

每条新闻右下角有点赞/踩按钮，用于收集读者对内容质量的反馈，长期影响选题权重。

当前实现（v0.3）：纯前端 localStorage

// 点击时存储
localStorage.setItem('thailand10_feedback', JSON.stringify({
  "a0": "up",    // 第1条：点赞
  "a3": "down",  // 第4条：踩
}))

// 再次点击同一按钮 → 取消投票
delete fb[article_id]

// 页面加载时恢复按钮状态
⚠️ 当前局限
数据存在本机浏览器里，Sub-agent 读不到，换设备后丢失。权重机制目前还没有真正接通。这是整个系统唯一未打通的闭环。

权重设计（已定义，待接通）

  • 👍 / 👎 计数独立累积，存储在新闻库条目的 popularity 字段
  • 仅在重要性相近（同 P 级）的候选稿件之间作为加权 tiebreaker
  • 不覆盖 AI 对新闻重要性的判断，编辑判断永远第一
  • 长期高 👍 的话题类型，逐步提高同类搜索关键词的优先级

待实现的数据回路

08 发布层：GitHub Pages

thailand10/ 整个目录是一个 GitHub 仓库（JiuTing6/thailand10）。每次 git push 后，GitHub 自动检测变化，重新构建静态站点，约 1–2 分钟后新内容全球上线。

URL 结构内容
/主页（双刊导航）
/thailand10/index.html期刊归档列表
/thailand10/YYYY-MM-DD.html每期正文
/docs/tech-workflow.html本技术文档
✅ 为什么选 GitHub Pages
全球 CDN、永久免费、版本控制、自动部署、无需维护服务器。唯一限制是静态，但当前架构完全不需要后端。
